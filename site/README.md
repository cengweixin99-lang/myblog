# Static Site Notes

This directory is the Zola site for the GitHub-issues-driven blog.

## What is source vs generated

- `templates/*.html`
  Tera templates that define the page structure.
- `static/*`
  Source static assets such as `site.css`.
- `content/_index.md`
  Home section config, including pagination.
- `content/pages/*`
  Generated special pages like `About`.
- `content/tags/*`
  Generated tag collection pages like `Top` or `Published`.
- `content/*.md`
  Generated post files, one authored issue per file.
- `public/*`
  Build output generated by Zola. Do not edit by hand.

## Content rules

- Any issue authored by `GITHUB_AUTHOR` or `GITHUB_OWNER` becomes a post.
- `About`, `Things I like`, and `Things I don't like` are reserved labels for standalone pages.
- `Top`, `Published`, and any other normal labels remain visible as post tags.
- Each visible tag gets its own collection page under `/tag/<slug>/`.

## Local workflow

1. Create a root `.env` from `../.env.example`.
2. Pull GitHub issues into generated Zola content:

```powershell
pnpm run fetch:issues
```

3. Build the site:

```powershell
zola build
```

4. Run a local preview server:

```powershell
zola serve
```

## Required environment variables

- `GITHUB_OWNER`
- `GITHUB_REPO`
- `GITHUB_TOKEN`
- `GITHUB_AUTHOR` (optional; defaults to `GITHUB_OWNER`)

## Generated directories

The fetch script recreates these every run:

- `content/pages/`
- `content/tags/`
- `content/*.md` generated from issues
